# 《制霸新手村的骷髅怪》商城系统策划案

> 版本: 1.0.5 | 更新日期: 2026-04-18 | 状态: 待开发

---

## 一、概述

### 1.1 目的
本策划案针对**微信小游戏备案**要求，设计商城系统最小可行方案。

### 1.2 商城定位
- **仅用于备案审核**：商城界面存在、商品展示完整、购买流程可演示
- **不接入真实支付**：点击购买后提示"功能暂未开放"
- **后续可扩展**：预留微信支付接口，待具备商户资质后可快速接入

### 1.3 设计原则
1. 复用现有UI组件和设计语言
2. 界面风格与现有主界面保持一致
3. 交互流程简单直接

## 二、商城入口设计

### 2.1 方案选择

**方案A：新增独立Tab**（推荐）
- 底部导航新增"商城"Tab
- 共6个Tab：主角、关卡、天赋、排行、世界、商城
- 优点：入口明显，符合用户预期

**方案B：入口按钮**
- 在"主角"Tab内增加"商店"入口
- 共5个Tab + 1个悬浮按钮
- 优点：不增加导航复杂度

### 2.2 推荐方案

采用**方案A：新增独立Tab**

```
导航栏图标：🎮 🎯 ⭐ 🏆 🗺️ 🛒
导航栏名称：主角 关卡 天赋 排行 世界 商城
```

## 五、功能流程

### 5.1 打开商城
```
用户点击"商城"Tab
    ↓
显示商城页面
    ↓
默认展示"金币"分类
```

### 5.2 购买商品流程
```
用户点击商品卡片
    ↓
弹出购买确认弹窗
    ↓
用户点击"确认购买"
    ↓
调用 wx.requestPayment() 或显示"功能暂未开放"
    ↓
（如果已接入支付）完成支付
    ↓
增加对应货币/道具到玩家账户
    ↓
显示购买成功提示
```

### 5.3 分类切换
```
用户点击分类标签
    ↓
切换商品列表（不刷新页面）
    ↓
平滑过渡动画
```
